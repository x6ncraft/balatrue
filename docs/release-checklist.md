## 两条发布线

源码公开与网站上线分别验收：

- **源码公开：**项目原创的功能定义、游戏逻辑、UI 与工程代码采用 MIT；公开文件树同时保留明确
  排除 MIT 的低分辨率卡图、逐卡来源记录和完整英文来源审查快照。
- **网站上线：**在源码边界之外，还要核对域名、托管、隐私、实网质量、中国大陆内容分类和卡图
  公开展示的残余权利风险。

`package.json` 的 `private: true` 只防止误发 npm，不是 GitHub 可见性门槛。

### 为什么可以直接公开现有历史

早期提交曾把完整英文效果与解锁描述放在生成目录；当前结构把同一批文字移到 `data/upstream/`，
避免浏览器加载并集中说明来源。项目已经选择有意保存并公开这些资料，根 MIT 明确不覆盖它们，
当前 notices 同样覆盖历史版本。重写历史不再有分发目的，也不能消除原作文本与卡图本身的权利
风险。
