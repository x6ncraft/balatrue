# 发布检查

本文件只维护任何贡献者都能从公开仓库复现的发布要求。域名注册、实名认证、账号、账单、控制台
状态、临时预览地址、单次部署流水和实网运维记录由站点维护者在仓库之外管理。

## 构建产物

- [ ] 从干净 checkout 执行 `bun install --frozen-lockfile`、`bun run check` 和
      `bun run test:e2e`，全部通过。
- [ ] 生产构建只依赖已提交的锁文件、来源审查快照与本地素材，不在 CI 或用户浏览器中远程抓取
      Wiki 数据。
- [ ] `dist/` 包含 `LICENSE`、`LICENSE_SCOPE.md`、`ASSET_NOTICE.md`、150 条逐卡来源记录、
      `THIRD_PARTY_NOTICES.md` 和运行依赖许可证。
- [ ] `dist/` 不包含 `data/upstream/` 的完整英文效果与解锁描述、环境文件、密钥、账号资料或内部
      文档。
- [ ] 静态托管使用根目录 `edgeone.json` 中固定的 Node、Bun 锁文件安装、构建命令与 `dist` 输出；
      若改用其他平台，仍保持同一构建边界。

## 产品与浏览器

- [ ] 每日局、无尽局、搜索、六次猜测、胜负、刷新恢复、分享、战绩、线索字典和图鉴旅程通过。
- [ ] 中英文界面、游戏内牌名、拼音和拼音首字母搜索均通过。
- [ ] 320px、375px、430px、手机横屏、761–900px 与 1440px 无页面级横向溢出，长线索不被静默
      裁切。
- [ ] 触控设备的输入框和选择器计算字号不低于 `16px`；聚焦不触发 iOS 自动放大，双击操作不造成
      意外缩放，同时保留用户主动双指缩放。
- [ ] 短屏中的候选列表、图鉴确认页和所有弹层可独立滚动，主要按钮始终可达。
- [ ] 键盘焦点、弹层焦点约束、减少动画设置、图片降级与屏幕阅读语义均有效。
- [ ] 真实 iPhone 微信内置浏览器完成聚焦、软键盘、横竖屏、连续竞猜和图鉴验收；桌面自动化不能
      代替这项真机检查。
