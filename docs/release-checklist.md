  MIT 的低分辨率卡图和逐卡来源记录，不包含完整英文效果与解锁审计原文。
- **网站上线：**在源码边界之外，还要核对域名、托管、隐私、实网质量、中国大陆内容分类和卡图
  公开展示的残余权利风险。

`package.json` 的 `private: true` 只防止误发 npm，不是 GitHub 可见性门槛。

### 为什么还需要干净历史

现有 Git 历史曾在生成目录中提交完整英文效果与解锁原文。当前文件删除或 Git 忽略不会移除历史
对象，直接把现有仓库设为 public 会重新公开这些版本。卡图则是当前公开口径有意保留、明确排除
MIT 的识别素材，不是此次清理目标。

建议从当前已验收文件树创建无父提交或新公开仓库，再逐项验证。历史改写、创建新仓库、切换
GitHub 可见性和推送新根提交属于外部或破坏性动作，等待用户明确确认后执行。
