# 视觉与交互

## 方向

这是一个打开就能玩的轻量 Web 应用。信息层级先于主题装饰：玩家在首屏先看到一句规则、
搜索框、提交按钮和六次机会；《小丑牌》的气质只通过暗色牌桌底色、像素标题、卡面比例和
少量红蓝强调色体现。

页面不使用大面积霓虹、火焰、CRT 扫描线、赌场摆件、厚重拟物按钮或占据首屏的神秘卡。
GPT-Image-2 的早期探索用于确认这些边界，最终实现以清楚、紧凑和快速为准。

## 核心构图

1. 顶部：紧凑品牌、今日／练习模式、语言和帮助。
2. 规则：一句话说明如何出牌和缩小范围；颜色与箭头的完整解释留在玩法弹窗。
3. 开局：既可搜索心里的牌，也可从五张随机牌里点一张直接开始；「换一组」会完整替换随机牌。
   候选牌选中前，提交按钮保持中性；选中后才成为红色的「猜猜看」主操作。
4. 结果：猜测牌面和五项线索；桌面为表格，常见手机把牌面身份与五项线索压在同一排，超窄屏再把身份单独放到上方。
5. 结局：答案、游戏内中英文名称、完整五项属性、成绩和分享入口。

界面使用 `g8` 玩家线索词汇。棋盘以 7 个主效果大类、8 个触发时机大类和 7 个依赖条件大类为主；
主效果的完整机制、依赖条件的完整细项仍决定绿色，处于同一大类但细节不同则显示黄色。线索字典
展示大类覆盖数量和 `c11` 精确机制、事件、条件细项；图鉴用同一组大类筛选，并在单卡详情中保留
精确说明。主效果、时机和条件各回答一个问题，不把结果、事件和前置状态混在一列。

英文像素标题与中文副标题先按行盒居中，再为系统中文字体做轻微的字形光学补偿，不依赖不同字体
之间并不等价的基线。尚未产生猜测时，游戏卡片只保留自身外边框；头部和反馈区之间的分隔线在
反馈区出现后再显示。

## 移动端

- 375px 宽度下，小丑牌身份与稀有度、价格、主效果、触发时机、依赖条件固定为同一行六列，表头与数据同列对齐。
- 首屏顶栏保持一行；精简重复的玩法提示和装饰文案，优先留出搜索、提交与六次机会所需空间。
- 尚未提交猜测时不强调候选数量；开始猜测后再展示缩小范围的反馈。
- 五张快速开局牌始终同排，点牌即提交第一猜；搜索框仍保留给已经有目标的玩家，页面加载时不自动
  弹出移动端键盘。
- 棋盘以大类为主；黄色副文案只在增加推理信息时出现，分别说明猜测中吻合的细项、同类但不同的
  细项，或答案另有内容。组合数据原子显示为完整自然短语，不使用裸计数，也不展示未猜到的谜底
  细项。图鉴和线索字典保留完整机制与条件细项，且列头可直接打开对应字典章节。
- 900px 及以下把黄色解释放到该次猜测下方的全宽说明条，五个大类格仍保持同排；340px 及以下
  进一步使用清楚的短标签，避免英文长词把单次猜测过度撑高。线索全文不裁切，整页不得横向滚动。
- 小丑图鉴和线索字典使用 `100dvh` 全屏面板，关闭按钮始终可见，内容区域独立滚动。
- 图鉴首屏保留搜索、结果数和筛选开关；四项筛选默认收起，避免挤占卡片浏览空间。
- 结局的五项答案属性在移动端仍保持一排五格，便于和上方猜测逐列核对。
- 触控按钮保持足够高度，键盘焦点、Escape 关闭和关闭后焦点返回与桌面一致。
- 触控设备中的输入框和选择器保持至少 `16px` 计算字号，避免 iOS 聚焦时自动放大；页面抑制双击
  缩放，但保留用户主动双指缩放与二维查看能力。
- 视口保留浏览器默认的安全区避让并适配动态高度；横屏短视口中的图鉴确认页可以独立滚动，两个
  操作按钮都能触达。
- 横向溢出从布局源头消除，不用裁切页面内容来掩盖回归。

## 动画节奏

- 提交后，结果行在 150ms 内进入，五项线索以 40–50ms 间隔揭示。
- 单次反馈总时长控制在 300–500ms，连续猜测不需要等待装饰动画。
- 数字箭头只做轻微位移，完全命中只保留短促高亮。
- 胜负只用答案卡翻面和少量原创花色粒子，不遮挡下一步操作。
- 尊重 `prefers-reduced-motion`。

## 字体与声音

- 英文品牌和数字使用开源 `Silkscreen`。
- 中文正文使用系统字体栈，优先 `PingFang SC`、`Noto Sans CJK SC` 和 `Microsoft YaHei`；项目不
  额外打包中文字体。
- 首版不依赖声音传达状态；未来若增加音效，只使用原创短音，不使用原游戏音乐和音效。
