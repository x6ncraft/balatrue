# 产品与规则

## 一句话定义

每天用最多六次出牌机会，猜出藏在牌背后的《小丑牌》小丑卡。

## 名称

- 英文名：`Balatrue`
- 中文名：`猜丑牌`
- 英文副标题：`A fan-made Balatro guessing game`
- 中文副标题：`《小丑牌》粉丝猜谜游戏`

`Balatrue` 把 `Balatro` 与 `true` 合在一起：既能直接唤起《小丑牌》，也对应从真假线索
中找出正确答案。中文名用三个字直接说明动作，并借用「小丑牌」里的「丑牌」形成轻松的
文字游戏。「猜猜今天是哪张小丑牌」只作为今日局的无障碍区域名称，不占用默认视觉空间。
页面短说明为「Balatro 粉丝自制小游戏 · 免费、非商业 · 非官方出品」。

玩法灵感来自 [Wordle](https://www.nytimes.com/games/wordle/index.html)、
[汉兜（Handle）](https://github.com/antfu/handle)和 BLAST.tv 的
[Counter-Strikle](https://blast.tv/counter-strikle/daily)。Balatrue 沿用“多次猜测、逐项反馈、
逐步缩小范围”的骨架，并把待猜对象和线索体系换成小丑卡。

## 首版模式

### 今日谜牌

- 每个北京时间自然日共享一个答案。
- 最多六次出牌机会。
- 150 张答案按固定种子洗牌，一个周期内不重复；新数据版本不会改变当天答案。
- 本地保存当日进度、胜负、连胜、胜率与平均出牌数。

### 无尽牌局

- 使用本地洗牌袋，整轮 150 张目标不重复。
- 每局结束后可立即再开一局，不影响每日成绩。

## 五项线索

| 线索           | 玩家看到的内容                 | 反馈规则                                   |
| -------------- | ------------------------------ | ------------------------------------------ |
| 稀有度         | 普通、罕见、稀有、传奇         | 相同为绿；否则箭头指向答案                 |
| 基础价格／获取 | 商店价格；传奇显示灵魂牌       | 同价或同为灵魂牌为绿；箭头指向答案         |
| 主效果         | 7 个大类与猜测牌的完整机制细项 | 机制集合相同为绿；机制不同但大类有交集为黄 |
| 触发时机       | 8 个大类与猜测牌的完整触发事件 | 事件集合相同为绿；事件不同但大类有交集为黄 |
| 依赖条件       | 7 个大类与猜测牌的完整条件细项 | 条件集合相同为绿；条件不同但大类有交集为黄 |

稀有度按官方简中顺序：`普通 → 罕见 → 稀有 → 传奇`。传奇牌没有官方商店价格；比较方向时，
“灵魂牌”排在所有商店价格之上，但不换算成推测金额。

数据校验要求 150 张牌的完整五维比较签名全部唯一，避免出现仅靠线索无法区分的二选一。每个
主效果大类至少覆盖 10 张牌，每个触发时机大类至少覆盖 5 张牌，每个依赖条件大类至少覆盖 8 张牌；
精确机制、触发事件和条件细项用于判绿与消歧，但不会独立成为棋盘大类。

## 线索语义与版本模型

三列分类各自只回答一个问题：

- 主效果：这张牌主要做什么。生成、复制、改造及其结果对象都写在这里。
- 触发时机：效果在什么事件发生、结算或更新。
- 依赖条件：到达该时机后，还要读取或满足什么已有的牌、牌型、资源、栏位或进度。

随机概率不算依赖；“必须有空位”算依赖。生成出来或改造后的对象、持续规则作用于哪些牌或牌型，
都不回填到依赖条件。“无需额外条件”表示除触发事件外没有额外局面要求。

| 小丑牌     | 棋盘主效果   | 棋盘触发时机 | 棋盘依赖条件 | 格内完整细项                   |
| ---------- | ------------ | ------------ | ------------ | ------------------------------ |
| 证书       | 生成         | 回合交界     | 无需额外条件 | 生成蜡封牌；回合开始           |
| 大理石小丑 | 生成         | 盲注阶段     | 无需额外条件 | 生成石头牌；选择盲注           |
| 石头小丑   | 筹码         | 整手结算     | 单牌／牌组   | 已有石头牌和全牌组数量决定筹码 |
| 四指       | 规则／再触发 | 持续生效     | 无需额外条件 | 改变顺子与同花的构成张数       |
| 卡牌术士   | 生成         | 盲注阶段     | 小丑／消耗牌 | 生成塔罗牌；需要空消耗牌栏     |
| 烧焦小丑   | 改牌／资源   | 手牌动作     | 弃牌         | 升级首次弃牌所对应的牌型       |

### 主效果

棋盘把 31 个审计机制收进 7 个容易横向比较的大类：

| 大类         | 收纳的精确机制                                                     |
| ------------ | ------------------------------------------------------------------ |
| 筹码         | 增加筹码                                                           |
| +倍率        | 增加加法倍率                                                       |
| ×倍率        | 提供乘法倍率                                                       |
| 经济         | 产生、保留或节省金钱                                               |
| 生成         | 生成或复制扑克牌、小丑牌、消耗牌或标签                             |
| 改牌／资源   | 改造、升级或摧毁已有牌；改变手牌上限、出牌、弃牌或重掷等牌局资源   |
| 规则／再触发 | 再触发其他效果；改变牌型、身份、盲注、生存、复制、重复或概率等规则 |

棋盘以大类为主，颜色仍比较底层完整机制集合：机制完全相同为绿色；机制不同但大类有交集为黄色；
大类无交集为灰色。无论颜色如何，格内都在大类下完整显示猜测牌自身的机制。这样“生成蜡封牌”和“生成
石头牌”都以“生成”为大类，并分别显示“蜡封牌”或“石头牌”；玩家通过颜色判断整项关系，自行推断
具体关联。格内不会逐项标注吻合或不同，也不会透露谜底独有的机制。

### 触发时机

棋盘用下面 8 个大类提供横向推理方向；`c11` 的 22 个精确事件作为猜测牌自身的完整细项显示在
大类下。精确事件集合相同为绿色；集合不同但至少同属一个大类为黄色；大类无交集为灰色。精确事件
不再作为只对应一两张牌的顶层游戏类别。

| 大类             | 精确事件                                                     |
| ---------------- | ------------------------------------------------------------ |
| 持续生效         | 持续生效                                                     |
| 整手结算         | 整手计分、其他小丑结算                                       |
| 逐牌计分         | 单牌计分                                                     |
| 手牌动作         | 打出手牌、留在手牌、弃牌                                     |
| 消耗牌／牌组变化 | 使用消耗牌、加入牌组、卡牌摧毁                               |
| 盲注阶段         | 选择盲注、跳过盲注、击败盲注、未过盲注                       |
| 商店相关         | 商店中、打开补充包、跳过补充包、商店重掷、离开商店、出售卡牌 |
| 回合交界         | 回合开始、回合结束                                           |

### 依赖条件

依赖条件把 16 个来源条件族归入 7 个玩家大类：

| 大类         | 说明                                                   |
| ------------ | ------------------------------------------------------ |
| 单牌／牌组   | 单张牌或牌组的花色、点数、牌面状态、留手状态与牌组数量 |
| 出牌／牌型   | 本次出牌的张数与顺序、计分牌、出牌记录或组成的牌型     |
| 弃牌         | 弃牌张数、次数、先后顺序、剩余次数或历史               |
| 金钱／商店   | 现有金钱、出售价值、商店重掷或售牌记录                 |
| 小丑／消耗牌 | 其他小丑牌、消耗牌、相对位置、已有数量或对应栏位       |
| 盲注／回合   | 盲注类型、Boss 状态、跳过／击败记录或经过回合          |
| 无需额外条件 | 除触发时机外，不读取其他牌、资源或局面                 |

棋盘以大类为主；底层条件细项集合完全相同才是绿色。条件不同但大类有交集为黄色，大类无交集为
灰色。无论颜色如何，格内都完整展示猜测牌自身的条件细项，不逐项标注吻合或不同，也不说明谜底
另有什么。多个数据原子先还原成自然条件，例如“钢铁牌＋全牌组数量”显示为“全牌组中的
钢铁牌数量”。页面给出的“约 N 张仍符合线索”只按玩家已经看见的大类、完整细项、颜色和方向计算；
不可见的逐项比较信息不能成为隐藏筛选条件。

分类数据分成两个版本层，既保留精确判断，也控制玩家需要理解的顶层词汇：

- `c11` 是逐卡审计层，记录 31 个主效果机制、22 个精确触发事件和 16 个来源条件族。完整英文效果
  与解锁描述保留在仓库审查层，浏览器只读取玩法所需的分类与摘要。
- `g9` 是玩家比较层：棋盘显示 7 个主效果大类、8 个触发时机大类和 7 个依赖条件大类，并在每个
  颜色格内保留猜测牌自身的完整细项；主效果机制、精确触发事件与依赖条件细项均参与精确比较，
  颜色承担比较结果，不增加逐项判断文案。

牌局存档键同时记录数据分类版本与玩家线索版本。恢复 `c11:g8`、`c11:g7`、`c10:g7`、`g6` 及更早
牌局时，应用保留答案、猜牌顺序、胜负和图鉴辅助状态，用当前分类重新计算每次反馈；只有重放后
牌局身份与状态一致，才写入新的 `c11:g9` 键。旧颜色不会直接沿用。变更玩家线索时必须重新检查
分类覆盖门槛、存档重算和 150/150 五维签名唯一性。

## 线索字典与小丑图鉴

- 棋盘以 7 个主效果大类、8 个触发时机大类和 7 个依赖条件大类为主；棋盘、图鉴筛选和字典始终
  使用同一套分类名称。三项分类线索都展示猜测牌自身的完整细项；绿、黄、灰只表达整项比较结果，
  不逐项判断，也不透露谜底细项。
- 五个线索列头都可打开字典并直接定位到对应章节；常驻的“线索字典”入口保留。
- 线索字典按上述大类解释规则，同时列出当前 150 张牌实际使用的精确机制、事件、条件细项及覆盖
  数量。它属于规则说明，随时查看都不影响成绩。
- 小丑图鉴包含 150 张牌的游戏内中英文名、卡图、稀有度、基础价格，以及完整主效果机制、精确
  触发事件和条件细项；四项筛选分别使用稀有度、7 个主效果大类、8 个时机大类和 7 个条件大类。
- 图鉴关闭后再次打开，保留本次页面会话中的搜索和筛选；刷新页面后恢复默认，并提供一键清空。
- 每日谜牌进行中打开图鉴前必须确认；确认后牌局仍可继续、保存和分享，但不写入已玩、胜率、连胜与平均出牌数。
- 已结束的每日局查看图鉴不会追溯取消成绩；无尽牌局本来就不计战绩，因此直接打开。
- 图鉴辅助局的分享首行带 `🔎` 标记，避免和独立完成的计分局混淆。

## 输入联想

- 同时索引游戏内简中名、英文名、中文全拼和拼音首字母。
- 中文界面以中文名为主、英文名为辅；英文界面反向展示。
- 候选项展示全部匹配结果和缩略图，不提前泄露稀有度和效果；列表限制可见高度并在内部滚动。
- 支持方向键、Enter、Escape 和中文输入法 composition，已经猜过的牌不可重复提交。
- 首局可搜索任意牌，也可点击随机展示的五张牌直接提交第一猜；「换一组」会换掉当前全部五张。
- 候选牌选中后，提交按钮才进入红色主操作状态；快速开局牌点下即提交，不再多一步确认。

卡图地址带本地内容摘要作为版本，首次加载失败会用新地址自动重试一次；仍失败时显示统一卡面占位，
牌名和玩法不受影响。

## 反馈和可达性

- `✓` 完全吻合：绿色。
- `◐` 部分吻合：黄色。
- 三种颜色格都展示猜测牌自身的大类与完整细项；颜色只表达整项比较结果，不逐项标注具体哪里吻合
  或不同，也不展示谜底细项。
- `×` 不吻合：暗红或中性灰。
- 数值不相同时，`↑ / ↓` 始终指向正确答案。
- 颜色之外始终保留图标、文本和边框差异。
