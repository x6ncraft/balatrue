# 部署、域名与基础统计

核验日期：2026-07-19。本文件记录会随时间变化的价格与平台政策；本轮没有购买域名、创建云资源
或部署站点。

## 建议结论

在用户确认素材残余风险，并完成对应地区的发布前置后，推荐：

- 产品主域：`balatrue.com`；短、通用，适合作为分享与搜索的唯一 canonical 地址。
- 品牌根域：`x6n.me`；保留 `balatrue.x6n.me` 作为 x6n 项目入口或跳转，后续其他“小玩具”也能
  使用同一结构。
- 托管：Tencent EdgeOne Makers，单个 Vite 静态站，服务区选择“全球含中国大陆”。
- 统计：首发只看 Makers 的请求、流量、地区和状态码；点击、开局和过关事件等后续再接 SDK。

子域名本身不另收注册费。普通 `.com` 的续费基准目前低于普通 `.me`，但 `x6n.me` 是三字符名称，
可能被注册局或注册商标为溢价／保留词；WHOIS 未见登记只说明当前未注册，不能据普通 `.me` 价
推算它的首年和续费。组合成本必须在购买页同时确认候选可用、是否溢价、首年价与续费价后再算。
若只买一个域名，`balatrue.com` 的成本更容易预估；如果准备持续经营 x6n，`x6n.me` 的复用价值
高于再囤多个 Balatrue 趣味后缀。

`handle.antfu.me` 的结构值得借鉴，但 Antfu 已经是成熟的个人品牌。Balatrue 当前更适合让产品名
占据最短主地址，同时用 `balatrue.x6n.me` 外露品牌。两个地址必须选一个 canonical，另一个做
永久跳转，避免分享和搜索权重分散。

## x6n 品牌域名

- `x6n.me` 在本次 RDAP／WHOIS 快照中未见登记，最终可用性和是否为溢价域名以下单页为准。
- `.me` 在当前中国大陆可备案后缀范围内；若要接入大陆节点，应在境内合规注册商注册并完成
  实名。子域名不产生新的注册费，备案与接入范围仍以服务商审核为准。
- `balatrue.x6n.me` 一般需要先完成域名所有权验证，再配置 CNAME；证书验证是否还需额外记录以
  EdgeOne 控制台实际流程为准。
- `x6n.com`、`x6n.cn` 当前已被注册。
- `x6n.x` 不可行。[IANA 根区数据库](https://www.iana.org/domains/root/db)没有公开的 `.x` 顶级域，
  因此它不能在公共互联网正常注册与解析。

若更看重极短，可以使用 `bt.x6n.me`，但口述时需要解释缩写；`balatrue.x6n.me` 自带产品名，
更适合作为长期地址。

## 域名候选与成本

与标准价，促销、会员价、溢价词、赎回价和最终结账价格都会变化。购买前必须在
[官方价格页](https://wanwang.aliyun.com/help/price.html)和候选域名的结账页重新核对。

| 候选                         |       首年／续费 | 大陆备案适配 | 维护判断                                       |
| ---------------------------- | ---------------: | ------------ | ---------------------------------------------- |
| `balatrue.com`               |   约 `¥83 / ¥90` | 当前支持     | 最通用、续费克制，首选产品主域                 |
| `x6n.me` → `balatrue.x6n.me` | `¥10–35 / ¥116`* | 当前支持     | 子域免费，品牌复用价值高；三字符可能溢价       |
| `balatrue.me`                |  `¥10–35 / ¥116` | 当前支持     | 短而轻松，但没有同时积累 x6n 品牌              |
| `balatrue.cn`                |   约 `¥35 / ¥39` | 支持         | 便宜，适合防御注册或跳转，不建议作全球唯一主域 |
| `balatrue.fun`               |  约 `¥12 / ¥175` | 当前支持     | 语气合适，但首年促销掩盖了较高续费             |
| `balatrue.games`             |   约 `¥50 / ¥88` | 当前支持     | 含义清楚，但更长且是复数                       |

\* 这是普通 `.me` 的展示区间，不是 `x6n.me` 报价。

本次快照中，`balatrue.com`、`balatrue.me`、`balatrue.cn`、`balatrue.fun`、`balatrue.games` 和
`x6n.me` 均未见登记。这不构成预留，域名可能随时被注册。

`.game` 的语义很好，但不在当前大陆备案后缀清单中，且注册商可能按高价后缀或溢价词计费；排除。
`.app`、`.dev`、`.io`、`.gg` 也不作为大陆主域候选。最终备案需同时满足后缀、注册服务机构、
实名主体和有效期要求，按[工信部域名管理机构公示](https://domain.miit.gov.cn/managerdns/index)与
接入商实时说明复核。

购买顺序仍分三档，但暂不写组合总价：

1. 只买 `balatrue.com`：成本最容易预估。
2. 再买 `x6n.me`：产品与品牌都保留，推荐先在结账页确认三字符是否溢价。
3. 确有防御注册需要时再加 `balatrue.cn`，不因首年便宜囤更多趣味后缀。

注册域名不会自动取得 `Balatrue` 名称或原作素材的权利。建议把拟用名称、域名和用途纳入权利
沟通；收到回复前仍不能写成默许或授权。购买时点由用户结合可用性、成本和残余风险决定。

## 托管方案

### 首选：EdgeOne Makers

[Makers 价格页](https://pages.edgeone.ai/pricing)当前承诺免费档长期可用，并列出 Git 部署、自动
CI/CD 与全球 CDN；具体项目和域名额度没有在当前公开价格页固定承诺，文档另列默认每月 500 次
构建。它可以连接 GitHub 仓库，生产分支 push 后自动构建；当前仓库的部署契约是：

```text
Install: bun install --frozen-lockfile
Build:   bun run build
Output:  dist
```

[自定义域名文档](https://pages.edgeone.ai/document/custom-domain)说明，选择中国大陆或全球含大陆
服务区时，域名必须先完成 ICP 备案。全球不含大陆可以不备案，但不能据此承诺中国内地访问质量；
Cloudflare 官方也明确其普通 Pages 不进入中国网络，因此不适合作为本项目的大陆主站。

Makers 的优势是一个站点覆盖全球、Git 发布简单、证书与 CDN 维护少。免费额度和产品政策仍可能
调整；保留阿里云 OSS + CDN 作为迁移预案即可，不必在首发同时维护两套生产链路。

### 大陆发布前置条件

ICP 备案解决域名接入，不等于网络游戏或网络出版分类已经解决。Balatrue 是无账号、无付费、
无下注的浏览器猜谜，但仍面向公众提供交互式游戏。启用大陆节点前，应提交玩法、截图、原作许可
范围和运营事实，取得属地主管部门或合规律师、托管平台的书面分类结论。

若书面结论允许按普通非经营性网站处理，再在境内注册商完成实名、ICP 和平台审核；网站上线后
按要求办理公安联网备案。若结论要求网络游戏出版与运营资质，当前个人静态站方案暂停。

## 基础统计

[Makers 项目指标](https://pages.edgeone.ai/document/metric-analysis)无需页面 SDK，可以查看请求量、
流量、带宽峰值、状态码与国家／地区。这能回答“有没有访问、来自哪里、是否有错误”，足够首发。

请求数不等于访客数：一次打开会请求 HTML、JavaScript、字体和多张图片。平台指标也不知道用户
是否开局、猜了几次、过关或点击外链。因此文档统一称“基础访问统计”，不称准确 PV／UV 或点击
统计。

后续确有需要时，再接一个无需自建服务端的前端事件 SDK，最小事件包括：

- `game_started`
- `guess_submitted`
- `game_finished`
- `collection_opened`
- `outbound_link_clicked`

接入前先更新页面隐私说明并提供适用的同意与撤回机制；不上传答案、具体猜测、自由文本和本地
存档。统计脚本加载失败时，游戏仍完整可玩。

## 执行顺序

1. 从干净公开候选 checkout 完成回归，由用户确认域名组合和素材残余风险。
2. 条件允许时继续询问名称、域名、卡图展示与源码再分发边界；未回复不写成授权。
3. 注册域名并建立稳定的权利／下架邮箱。
4. 取得中国大陆内容分类结论；若允许，再办理实名、ICP 与平台审核。
5. Makers 导入仓库，绑定主域和品牌跳转域，申请证书并强制 HTTPS。
6. 用中国电信、联通、移动和至少一个海外网络验收首屏、卡图、图鉴与分享。
7. 首发观察平台基础指标；真实需求出现后再决定前端事件统计。

外部动作均以 `release-checklist.md` 为准，用户确认前不执行。
