# 数据来源与边界

## 当前数据版本

玩法目标版本标记为《小丑牌》`1.0.1o-FULL`，共 150 张小丑牌：普通 61、罕见 64、稀有 20、
传奇 5。当前英文本地化与简中本地化镜像实际标记为 `1.0.1m-FULL`；现有机器没有可核对的
`1.0.1o-FULL` 游戏包，因此尚未完成两个版本的名称差异核对。文档和界面把它们称为“当前数据
版本中的游戏内名称”，不写成已经由 `o` 版游戏包直接验证。运行时不访问远程 API，也不在用户
浏览器中重新分类。

现有目录来自社区资料站 Balatro Wiki 的表格、图片和游戏本地化镜像；该站由 Weird Gloop 托管。
Wiki 在本项目中只承担资料与出处来源角色，不被写成 Balatro 原始权利人或卡图授权方。Balatro
由 LocalThunk 开发、Playstack 发行，相关名称、商标、游戏文本和卡图的权利归相应权利人所有。
以后取得当前游戏包时，先生成差异报告，再决定是否更新版本；不自动覆盖人工分类。

## 四类数据产物

`bun run data:sync` 在维护者明确允许远程同步后生成四种用途不同的产物：

- `src/data/jokers.generated.ts`：随仓库提交，只保留浏览器玩法需要的双语名称、稀有度、价格、
  分类、图片摘要和来源文本 SHA-256。
- `src/search/joker-search.generated.ts`：随仓库提交，只保留由上述简中名称生成的无声调全拼和拼音
  首字母；可用 `bun run data:search` 从当前目录离线重建。
- `data/jokers.provenance.generated.json`：随仓库提交，逐卡记录资料页、图片文件说明页、图片来源
  URL、来源与本地图片摘要、尺寸、名称、来源文本摘要和固定修订。它用于追溯，不代表取得卡图
  或文本授权。
- `data/upstream/jokers.wiki.generated.json`：随仓库提交，包含价格、稀有度、图片元数据、完整英文
  效果、解锁条件和其他离线生成输入，用于复核、追溯和后续升级。它属于第三方来源审查资料，
  不适用项目 MIT 许可证，也不进入浏览器构建；来源与许可声明见同目录 `README.md`。

标准安装、`bun run check`、E2E 和生产构建依赖这四份已提交产物；`bun run data:generated:check`
会用当前纯转换逻辑与本地图片离线重建运行目录、检索词和逐卡来源记录，再深比较已提交产物。
数据校验还会重算 300 个原文摘要并核对业务不变量。`bun run data:provenance` 用于维护者从来源
审查数据重新生成逐卡来源记录，不属于普通构建步骤。生产 `dist/` 只包含运行目录、检索词与来源
记录，不包含完整英文描述。

## 来源入口

- 小丑表格：MediaWiki `action=parse&page=Jokers`；metadata 记录实际
  `Jokers?oldid=<revision>`。该 `oldid` 固定容器页，但表格通过 DPL 动态汇总各小丑页字段，不能
  代替 150 个单页各自的修订号；现有快照依靠逐卡 URL、保存内容和摘要追溯。
- 游戏英文本地化镜像：`Module:Localization/en-us`；metadata 记录固定 `oldid`。
- 游戏简中本地化镜像：`Module:Localization/zh-cn`；metadata 记录固定 `oldid`。
- 图片：MediaWiki 图片分类 API；下载到 `public/jokers/` 后本地使用。

具体 URL、解析和分类逻辑只由 `scripts/sync-jokers.ts` 维护。生成文件不得手改。

Wiki 运营方条款对自动化访问另有要求。远程同步只供维护者在确认访问条件后执行：

```bash
BALATRUE_REMOTE_SYNC_ALLOWED=1 bun run data:sync
```

这个环境变量只表示维护者已完成前置确认，本身不会产生访问或内容许可。同步不能进入 CI；正常
安装、测试与生产构建均不联网取数。

## Wiki 文字与卡图边界

Balatro Wiki 当前页脚标注 CC BY-NC-SA 3.0，但该站由标注 CC BY-SA 的 Fandom Wiki 迁出，迁移
前后文字许可如何衔接尚未确认；部分短文案还可能来自游戏原文。项目选择把完整英文描述作为
有意公开、可复核的上游快照保存在 `data/upstream/`，同时标记 `NOASSERTION`、明确排除 MIT，并
保留 Wiki 贡献者、迁移沿革、原作权利和下架说明。项目不以自己的名义重新许可这些文字，也不
声称一种 CC 许可证覆盖所有字段。

Wiki 的文字许可不自动覆盖非文字文件。公开来源记录为 150 张卡图保留逐卡资料页、文件说明页、
URL 和摘要，但这些记录是出处证据，不是作者或许可证明。项目把低分辨率卡图保留为当前不用于
盈利的免费、非商业粉丝猜谜牌面识别材料，并在 `ASSET_NOTICE.md` 中排除 MIT、说明非官方身份和
下架方式；这些措施不构成授权，公开展示与再分发仍有残余权利风险。如有侵权疑问，优先立即下线
相关素材，再核对、移除或替换。

## 分类版本

当前使用**能力数据模型第 12 版**（内部存档标识 `c12`）：每张牌按独立能力子句记录触发事件、
能力动作、产生效果、事件筛选、外部读取和自身门槛，并保留获取方式和按稳定 ID 维护的逐卡审计修订。
`CLASSIFICATION_OVERRIDES` 是人工修订的唯一入口；
`bun run data:regenerate` 可从已提交的来源审查快照离线重建运行目录和来源记录。

当前使用**玩家线索规则第 11 版**（内部存档标识 `g11`）：

- 主效果投影为筹码、+倍率、×倍率、金钱、生成／复制、改牌／资源、规则／再触发 7 个大类；
  成长、递减、重置、换目标和自毁作为完整细项保留。
- 精确触发事件投影为持续生效、整手计分、逐牌计分、卡牌动作、盲注阶段、商店／补充包、
  回合开始／结束 7 个玩家阶段；精确事件本身不参与该列比较。
- 判定细项投影为单牌／牌组、出牌／牌型、弃牌、金钱／商店、小丑／消耗牌、盲注／进度、
  无需额外判定 7 个大类。通用空栏限制不进入玩家线索；多动作阶段在没有具体条件时补入真实动作。

当前玩家线索规则延续特殊获取方式的比较顺序：传奇牌仍保留来源中的 `null` 商店价格，界面显示
“灵魂牌”；方向提示把它排在所有商店价格之上，但不把出售价格反推成官方购买价格。主效果与判定
依据分别以完整机制和完整判定细项集合判绿，触发时机以玩家阶段集合判绿；集合不同但玩家大类有
交集为黄色，大类无交集为灰色。主效果与判定依据格展示猜测牌自身的大类与完整细项，时机格只展示
阶段；颜色只表达整项完全、部分或不吻合。格内不逐项指出具体哪里吻合，也不透露谜底独有的内容。
组合判定先还原为自然短语。

能力子句拆解、三组玩家大类、判定边界和重点牌回归范围统一由
[`clue-taxonomy.md`](clue-taxonomy.md) 维护。分类变更必须递增相应版本，重新生成并人工复核
`classification-review.generated.md`，再验证分类覆盖门槛、`c12:g10 → c12:g11` 及更早存档重算和
五维比较签名唯一性。生成物、转化结果和普通自身累计值不得误作判定依据；精确事件只在需要区分
多动作时投影成玩家可见动作。

## 构建校验

- 总数为 150；ID、图鉴编号和中英文名称唯一。
- 稀有度分布为 `61 / 64 / 20 / 5`。
- 每张牌有价格或明确的特殊获取方式，以及可核对的本地图片摘要。
- 150 条逐卡来源记录与运行目录的名称、摘要、尺寸和来源修订一一对应。
- 150 条来源审查记录齐全，300 个英文原文字段与运行目录摘要一致。
- 运行目录的价格、稀有度、分类、图片元数据，检索词和公开来源记录与当前来源快照及转换逻辑逐项
  一致。
- 所有玩家可见分类值都有自然的中英文文案。
- 当前能力数据中的每个主效果、精确触发事件和条件细项都能投影为玩家线索；所有玩家可见结果都有
  自然的中英文说明。
- 7 个主效果大类各覆盖至少 10 张牌；7 个时机大类各覆盖至少 5 张牌；7 个判定依据大类各覆盖至少
  6 张牌。
- 生成的分类复核表覆盖规范中的全部代表牌与历史调整牌，以一张表列出与棋盘一致的玩家侧最终
  五维，并已逐张人工确认；完整能力子句由分类数据和自动化测试单独校验。
- 证书、大理石小丑等生成牌不把生成结果误作判定；石头小丑等读取既有牌组状态的牌仍保留真实依据。
- 150 张牌在当前玩家线索规则下的完整五维比较签名为 `150/150`，没有仅靠比较规则无法区分的答案。
- 生产构建包含项目许可证、素材说明和公开来源记录，并拒绝仓库审查用完整原文。

卡图、游戏内名称、Wiki 文本和自动访问分别有自己的权利边界。这里记录生成事实；发布判断见
`legal-notice.md` 和 `release-checklist.md`。
