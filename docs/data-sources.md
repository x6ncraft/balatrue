# 数据来源与边界

## 当前数据版本

玩法目标版本标记为《小丑牌》`1.0.1o-FULL`，共 150 张小丑牌：普通 61、罕见 64、稀有 20、
传奇 5。当前英文本地化与简中本地化镜像实际标记为 `1.0.1m-FULL`；现有机器没有可核对的
`1.0.1o-FULL` 游戏包，因此尚未完成两个版本的名称差异核对。文档和界面把它们称为“当前数据
版本中的游戏内名称”，不写成已经由 `o` 版游戏包直接验证。运行时不访问远程 API，也不在用户
浏览器中重新分类。

现有目录来自社区资料站 Balatro Wiki 的表格、图片和游戏本地化镜像；该站由 Weird Gloop 托管。
Wiki 在本项目中只承担资料与出处来源角色，不被写成 Balatro 原始权利人或卡图授权方。Balatro
由 LocalThunk 开发、Playstack 发行，相关名称、商标、游戏文本和卡图的权利归相应权利人所有。
以后取得当前游戏包时，先生成差异报告，再决定是否更新版本；不自动覆盖人工分类。

## 三层数据产物

`scripts/sync-jokers.ts` 在维护者明确允许远程同步后生成三种用途不同的产物：

- `src/data/jokers.generated.ts`：随仓库提交，只保留浏览器玩法需要的双语名称、稀有度、价格、
  分类、图片摘要和来源文本 SHA-256。
- `data/jokers.provenance.generated.json`：随仓库提交，逐卡记录资料页、图片文件说明页、图片来源
  URL、来源与本地图片摘要、尺寸、名称、来源文本摘要和固定修订。它用于追溯，不代表取得卡图
  或文本授权。
- `data/upstream/jokers.wiki.generated.json`：随仓库提交，包含完整英文效果、解锁条件和解析所需
  的来源字段，用于复核、追溯和后续升级。它属于第三方来源审查资料，不适用项目 MIT 许可证，
  也不进入浏览器构建；来源与许可声明见同目录 `README.md`。

标准安装、`bun run check`、E2E 和生产构建依赖这三份已提交产物；数据校验会重新计算 300 个原文
摘要并核对运行目录。`bun run data:provenance` 用于维护者从来源审查数据重新生成逐卡来源记录，
不属于普通构建步骤。生产 `dist/` 只包含运行目录与来源记录，不包含完整英文描述。

## 来源入口

- 小丑表格：MediaWiki `action=parse&page=Jokers`；metadata 记录实际
  `Jokers?oldid=<revision>`。该 `oldid` 固定容器页，但表格通过 DPL 动态汇总各小丑页字段，不能
  代替 150 个单页各自的修订号；现有快照依靠逐卡 URL、保存内容和摘要追溯。
- 游戏英文本地化镜像：`Module:Localization/en-us`；metadata 记录固定 `oldid`。
- 游戏简中本地化镜像：`Module:Localization/zh-cn`；metadata 记录固定 `oldid`。
- 图片：MediaWiki 图片分类 API；下载到 `public/jokers/` 后本地使用。

具体 URL、解析和分类逻辑只由 `scripts/sync-jokers.ts` 维护。生成文件不得手改。

Wiki 运营方条款对自动化访问另有要求。远程同步只供维护者在确认访问条件后执行：

```bash
BALATRUE_REMOTE_SYNC_ALLOWED=1 bun run data:sync
```

这个环境变量只表示维护者已完成前置确认，本身不会产生访问或内容许可。同步不能进入 CI；正常
安装、测试与生产构建均不联网取数。

## Wiki 文字与卡图边界

Balatro Wiki 当前页脚标注 CC BY-NC-SA 3.0，但该站由标注 CC BY-SA 的 Fandom Wiki 迁出，迁移
前后文字许可如何衔接尚未确认；部分短文案还可能来自游戏原文。项目选择把完整英文描述作为
有意公开、可复核的上游快照保存在 `data/upstream/`，同时标记 `NOASSERTION`、明确排除 MIT，并
保留 Wiki 贡献者、迁移沿革、原作权利和下架说明。项目不以自己的名义重新许可这些文字，也不
声称一种 CC 许可证覆盖所有字段。

Wiki 的文字许可不自动覆盖非文字文件。公开来源记录为 150 张卡图保留逐卡资料页、文件说明页、
URL 和摘要，但这些记录是出处证据，不是作者或许可证明。项目把低分辨率卡图保留为当前不用于
盈利的免费、非商业粉丝猜谜牌面识别材料，并在 `ASSET_NOTICE.md` 中排除 MIT、说明非官方身份和
下架方式；这些措施不构成授权，公开展示与再分发仍有残余权利风险。如有侵权疑问，优先立即下线
相关素材，再核对、移除或替换。

## 分类版本

当前原始分类版本是 `c9`：18 个触发事件、16 个依赖族，以及主效果、获取方式和少量按稳定 ID
维护的人工修订。`CLASSIFICATION_OVERRIDES` 是人工修订的唯一入口。

当前玩家线索版本是 `g3`：

- 11 个触发时机：常驻、出牌时、留在手牌、弃牌时、使用消耗牌时、加入牌组时、卡牌摧毁时、
  盲注阶段、商店／出售、回合开始、回合结束。
- 8 个依赖条件：扑克牌／牌组、牌型、出牌／弃牌、金钱／商店、小丑／栏位、消耗牌、盲注／回合、
  无特定条件。

`g3` 把相近原始触发事件合并后比较；依赖条件保留来源细项，完整细项集合相同才是绿色，同组但
细项不同可以是黄色。分类变更必须递增相应版本，并重新验证存档兼容和五维签名唯一性。

## 构建校验

- 总数为 150；ID、图鉴编号和中英文名称唯一。
- 稀有度分布为 `61 / 64 / 20 / 5`。
- 每张牌有价格或明确的特殊获取方式，以及可核对的本地图片摘要。
- 150 条逐卡来源记录与运行目录的名称、摘要、尺寸和来源修订一一对应。
- 150 条来源审查记录齐全，300 个英文原文字段与运行目录摘要一致。
- 所有玩家可见分类值都有自然的中英文文案。
- `c9` 每个触发事件都能投影到 `g3`，玩家看到的 11 个触发时机和 8 个依赖条件均有定义。
- 150 张牌在 `g3` 下的完整五维签名互不重复。
- 生产构建包含项目许可证、素材说明和公开来源记录，并拒绝仓库审查用完整原文。

卡图、游戏内名称、Wiki 文本和自动访问分别有自己的权利边界。这里记录生成事实；发布判断见
`legal-notice.md` 和 `release-checklist.md`。
