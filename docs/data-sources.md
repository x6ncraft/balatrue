# 数据来源与边界

## 数据版本

首版锁定《小丑牌》`1.0.1o-FULL` 的 150 张小丑牌：普通 61、罕见 64、稀有 20、
传奇 5。数据生成日期和人工分类版本写入生成结果，不在运行时访问远程 API。

## 权威顺序

1. 用户已购游戏包中的 `localization/en-us.lua`、`localization/zh_CN.lua`、小丑定义和贴图。
2. Balatro Wiki 的结构化模块与 150 张表格，用于当前机器没有游戏包时生成原型和交叉核验。
3. 项目人工分类：主效果、生效时点与依赖条件；这部分是竞猜模型，不冒充官方事实。

当前机器未安装游戏，因此首版由 Wiki 镜像生成，并记录来源 URL、内容版本和图片 SHA-1。
用户提供当前游戏包后，应运行差异检查，以游戏包为最终事实。

## 自动化入口

- 小丑结构化数据：`Module:Jokers/data`
- 官方英文本地化镜像：`Module:Localization/en-us`
- 官方简中本地化镜像：`Module:Localization/zh-cn`
- 150 张表格：MediaWiki `action=parse&page=Jokers`
- 图片：MediaWiki 图片分类 API；下载到 `public/jokers/` 后本地使用

具体 URL 和解析逻辑由 `scripts/sync-jokers.ts` 统一拥有，生成结果不得手改。

## 构建校验

- 总数必须为 150；ID、图鉴编号、英中名称必须唯一。
- 英中本地化 key 集完全一致。
- 稀有度分布固定为 `61 / 64 / 20 / 5`。
- 每张牌必须有本地图片、价格或明确的特殊获取方式。
- 所有玩家可见分类值必须有中英文文案。
- 下一版游戏数据先生成 diff，不自动覆盖人工分类。

## 版权

社区 Wiki 的页面许可证不能被解释为对游戏美术的再授权。官方 Press Kit 也没有提供全部
150 张独立卡图的互动产品授权。当前卡图只进入私人、非商业原型。公开部署或开源包含卡图
之前，按 `docs/blockers.md` 取得许可或替换素材；代码与官方卡图保持可分离。
