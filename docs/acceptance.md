# 验收标准

## 必须能玩的链路

1. 打开页面，不读帮助也能从一句规则、搜索框和八次机会理解玩法。
2. 用中文、英文或拼音搜索任意小丑，支持键盘选中并提交。
3. 每次猜测出现真实牌图和五项线索；颜色、图标、文字与箭头一致。
4. 刷新后恢复当日进度；猜中或八次用尽后正确揭示答案。
5. 切换中英文后，官方名称与全部 UI 文案同步切换。
6. 能开始无尽牌局，且不覆盖今日成绩。
7. 能复制不泄露答案的分享结果。

## 技术回归

- `bun run check` 全部通过。
- 数据数量、唯一性、稀有度分布、图片存在性和枚举合法性通过自动校验。
- 反馈算法覆盖完全命中、同族部分命中、无命中和数值箭头。
- 每日答案算法在同一天稳定，跨天变化，一个 150 张周期内不重复。
- Playwright 覆盖桌面、短屏移动端、拼音与输入法 composition、键盘联想、重复猜测、胜负、刷新恢复、战绩、弹层焦点、语言切换与提交后反馈可见性。

## 视觉与可达性

- 375px、短屏移动端和 1440px 桌面端无水平页面溢出。
- 长英文名、长中文名、空搜索、重复猜测都有明确状态；卡图缺失时仍保留双语名称。
- 键盘焦点可见；输入与弹层语义可由辅助技术读取。
- 不只依赖颜色表达结果；减少动画设置有效。
- 自动化旅程中浏览器控制台无错误；公开部署前再执行 Lighthouse 基线审查。
