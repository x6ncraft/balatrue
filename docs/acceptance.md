# 验收标准

## 必须能玩的链路

1. 打开页面，不读帮助也能从一句规则、搜索框和六次机会理解玩法。
2. 用中文、英文或拼音搜索任意小丑；全部匹配项都能在可滚动候选列表中找到，支持键盘选中并
   提交。
3. 每次猜测出现真实牌图和五项线索；颜色、图标、文字与箭头一致，数值与方向箭头保持同行。
4. 刷新后恢复当日进度；猜中或六次用尽后正确揭示答案，并列出答案的完整五项属性。
5. 切换中英文后，当前数据版本中的游戏内名称与全部 UI 文案同步切换。
6. 能开始无尽牌局，且不覆盖今日成绩。
7. 能复制不泄露答案的分享结果。
8. 线索字典完整解释五项线索，打开、关闭和刷新都不改变计分资格。
9. 每日局打开图鉴前明确提示；确认后能搜索、筛选 150 张牌，并在刷新后保持“图鉴辅助局、不计战绩”状态。
10. 图鉴在同一页面会话中关闭再开会保留搜索与筛选，一键清空后恢复全部 150 张；刷新页面后不
    保留临时筛选。

## 技术回归

- `bun run check` 全部通过。
- 干净 checkout 不含本地受限审计 JSON，仍能完成安装、`bun run check` 与 E2E。
- 生产构建包含项目 MIT 许可证、`ASSET_NOTICE.md`、150 条逐卡来源记录、
  `THIRD_PARTY_NOTICES.md` 和运行依赖的原始许可证文本，也不包含本地受限字段。
- 数据数量、唯一性、稀有度分布、图片存在性和枚举合法性通过自动校验。
- 公开来源记录与运行目录的 ID、名称、来源摘要、图片摘要和尺寸一一对应。
- `c9` 原始审计分类可稳定投影为 `g3` 的 11 个“触发时机”和 8 个“依赖条件”；两个版本都进入存档兼容边界。
- `g3` 公开五维签名在 150 张牌中仍然唯一。
- 反馈算法覆盖完全命中、同族部分命中、无命中和数值箭头。
- 每日答案算法在同一天稳定，跨天变化，一个 150 张周期内不重复。
- Playwright 覆盖桌面、短屏移动端、拼音与输入法 composition、连续键盘竞猜焦点、重复猜测、胜负、刷新恢复、存储失败、战绩、弹层焦点、语言切换、五类字典、图鉴不计分、筛选折叠与提交后反馈可见性。

## 视觉与可达性

- 320px、375px、430px、761–900px 中宽度、短屏移动端和 1440px 桌面端无水平页面溢出。
- 常见手机宽度下，每次猜测的牌面身份与五项反馈位于同一行，跨猜测保持列对齐。
- 340px 及以下使用“牌名独占一行＋五项同排”的可读性兜底；所有宽度下长依赖都不被静默截断。
- 移动端初始视图不因顶栏换行、重复规则或空白结果区挤压搜索与提交操作。
- 结局卡在桌面和移动端都展示答案的稀有度、价格／获取、主效果、触发时机和依赖条件。
- 长英文名、长中文名、空搜索、重复猜测都有明确状态；卡图缺失时仍保留双语名称。
- 键盘焦点可见；输入与弹层语义可由辅助技术读取。
- 不只依赖颜色表达结果；减少动画设置有效。
- 自动化旅程中浏览器控制台无错误；公开部署前再执行 Lighthouse 基线审查。

## 发布候选基线

2026-07-19 的发布候选已建立以下回归基线：

- `bun run check` 覆盖格式、类型、单元测试、ESLint、数据校验和生产构建。
- `bun run test:e2e` 覆盖 1440px 桌面与移动 Chromium 的核心旅程；移动专用旅程在桌面项目按设计跳过。
- 数据校验：150 张游戏内中英文名称、150 张本地图片、150 条公开来源记录和
  `61 / 64 / 20 / 5` 稀有度分布通过；本地审计文件存在时追加完整原文摘要核对。

每次交付仍需记录当次命令结果，并在真实浏览器复核 320×568、375×667、430px、761px、
800px、900px 与 1440×900：牌面与五项线索六列同行、长线索不裁切、图鉴／字典全屏布局和无水平溢出均
通过，刷新后新增控制台错误为 0。

源码公开前还要从当前文件树建立不含历史完整原文的干净根提交或新仓库，并从该 checkout 重跑
上述回归；历史改写、创建仓库与切换可见性等待用户确认。卡图继续作为明确排除 MIT 的非商业粉丝
识别素材，逐卡来源、免责声明和下架机制不得写成已经授权。公开部署另需核对域名、托管、隐私与
残余权利风险；启用中国大陆节点前取得内容分类书面结论。
