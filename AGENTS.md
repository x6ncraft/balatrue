# Balatrue 协作规范

作用范围：整个仓库。

## 产品不变量

- 玩家可见名称必须使用当前游戏版本的官方英文与简体中文本地化。
- 每日答案在同一北京时间自然日内稳定；数据更新不能改变当天已经开始的牌局。
- 反馈算法只读取版本化的公开字段，不根据答案名称或图片做隐式特判。
- 绿色表示完全吻合，黄色表示同族或集合部分吻合，错误状态使用图标和文字辅助颜色。
- 纯前端答案可以被检查，因此不承诺防作弊，不设计有奖排名。

## 交付节奏

- 按可独立运行、可独立验收的模块推进，并使用简明的 Conventional Commit。
- 数据事实、人工分类和 UI 翻译分别维护，人工分类必须有版本和可审计定义。
- 新功能同时更新相邻测试和最窄权威文档。
- `bun run check` 是完整技术回归入口；交付还必须通过真实浏览器试玩。

## 常用命令

```bash
bun install
bun run data:sync
bun run dev
bun run test
bun run test:e2e
bun run check
```

## 素材边界

- 官方卡图仅用于当前私人原型，不自动随未来开源代码发布。
- 不加入原游戏音乐、Logo、完整源码或游戏包。
- 公开部署前按 `docs/blockers.md` 处理素材许可与替代方案。
